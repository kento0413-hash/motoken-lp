# Cycle 2 Analysis — アクセシビリティ・レスポンシブ対応

## ターゲット領域
アクセシビリティ・レスポンシブ対応: 6/10

## ユーザーの目
1. モバイルメニューを開いた時、フォーカスがメニュー内にトラップされていない → Tab操作でメニュー外に移動できてしまう
2. Worksセクションの外部リンクが新しいタブで開くことがスクリーンリーダーで伝わらない
3. Navのrole="menubar"/role="menuitem"はアプリケーションメニュー向けで、ナビゲーションリンクには不適切

## デザイナーの目
1. モバイル(480px以下)でWorksカードの最小高さが280pxだが、コンテンツに対してやや低い可能性
2. Pillarsグリッドが1024pxで1カラムに切り替わるが、768px-1024pxで2カラムにすると読みやすい

## エンジニアの目
1. role="menubar"/"menuitem"の誤用 — WAI-ARIAのmenubar roleはアプリケーションメニュー(File/Edit等)向け。サイトナビではrole不要でnav要素で十分
2. 外部リンクにaria-labelの補足がない
3. Statsセクションにid属性がないためランドマークとして不完全

## 実装計画（最大5件）
1. **Nav: role="menubar"/"menuitem"/"none"を削除** — nav要素で十分、不要なARIA roleは有害
2. **モバイルメニューにフォーカストラップ追加** — Tab操作でメニュー内を循環
3. **Worksの外部リンクにaria-labelで「新しいタブで開く」を補足** — スクリーンリーダーでの情報提供
4. **ContactのsocialLinkにも同様のaria-label補足** — 一貫性
5. **Statsセクションにidを追加** — ランドマーク完成
